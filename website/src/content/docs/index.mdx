## Why chord-gateway

<CardGrid>
  <Card title="Chat control plane" icon="external">
    Connect WeChat iLink or Feishu to `chord headless` and control Chord from an IM chat, including mobile devices.
  </Card>
  <Card title="Local-first boundary" icon="seti:lock">
    Chord processes, workspace files, runtime state, and credentials stay on your machine or server.
  </Card>
  <Card title="Per-chat isolation" icon="document">
    Gateway routes by workspace, IM type, and chat ID so different chats keep separate pinned Chord sessions.
  </Card>
  <Card title="Interactive approvals" icon="rocket">
    Confirmations, questions, todo updates, and session commands are surfaced back into the chat workflow.
  </Card>
</CardGrid>

## Get started

<CardGrid>
  <Card title="1. Install" icon="seti:go">
    ```bash
    go install github.com/keakon/chord-gateway@latest
    ```
    Install Chord separately and make sure `chord` is on `PATH`, or set `chord_path` in the gateway config.
  </Card>
  <Card title="2. Choose an IM platform" icon="random">
    Start with [WeChat iLink](/chord-gateway/wechat/) or [Feishu](/chord-gateway/feishu/), then keep only the adapter you need in `config.yaml`.
  </Card>
  <Card title="3. Run the gateway" icon="seti:powershell">
    ```bash
    chord-gateway -f config.yaml
    ```
    Send `/status` from the connected chat to verify routing and the `chord headless` child process.
  </Card>
</CardGrid>
